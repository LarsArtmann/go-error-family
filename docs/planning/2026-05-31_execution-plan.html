Mockable Diagnostic Helpers

Git (69.2%) and Postgres (58.6%) coverage is low because RunCommand and CommandExists are concrete functions shelling out to the OS. We need an injectable layer.

Design Decision: Introduce a CommandRunner interface in the diagnose package. Rules accept it as an optional field. Default behavior uses the existing RunCommand/CommandExists functions. What NOT to Do

  • No go-git migration — GitRule works fine with exec.Command; the dependency cost isn't justified for a diagnostic rule that runs once per error
  • No pkg/ restructure — the flat package layout is correct for a focused library
  • No Docker integration tests — mock-based testing achieves coverage targets without CI complexity
  • No coverage thresholds in CI yet — wait until coverage stabilizes at 80%+ across all packages
  • No auto-fix capability — the agent is analysis-only by design; adding execution violates the library's contract
  • No breaking changes to existing public API — all additions are additive; CommandRunner defaults to existing behavior
